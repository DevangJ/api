Introduction

The API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.
BASE URL
https://hosturl.com

Authentication

The API uses API keys to authenticate requests.

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

Authentication to the API is performed via HTTP Basic Auth. Provide your API key as the basic auth username value. You do not need to provide a password.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

AUTHENTICATED REQUEST
curl https://hosturl.com/sql \
-u default_key:

Errors

The API uses conventional HTTP response codes to indicate the success or failure of an API request. In general: Codes in the 2xx range indicate success. Codes in the 4xx range indicate an error that failed given the information provided (e.g., a required parameter was omitted, input was erronous, etc.). Codes in the 5xx range indicate an error with the API's servers (these are rare).

HTTP STATUS CODE SUMMARY
200 - OK
400 - Bad Request
405 - Not Allowed
Success
Invalid Parameters
Invalid Method Used

SQL Injection

Use this to sanitize input strings for sql query execution. input is given via query string. output is recieved in json format.

ENDPOINT
POST /sql

PARAMETERS
$ datatype(string)
Available values: integer, string
Type of Input
$ input(string)
Input String

RESPONSES
200
successful operation
Example response
                          {
                            "output" : "sanitized string" 
                          }
400
invalid json parameters
